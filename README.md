# Gearbox Product Docs Vault

This vault collects the core product thinking for Gearbox user research, decision flows, jobs-to-be-done, and open data requirements.

## Foundations

- [Basic info and definitions](Basic%20info%20and%20definitions.md) - Defines the key Gearbox concepts, yield sources, and the canonical `Discover -> Analyze -> Propose -> Preview -> Execute -> Monitor` loop.
- [Personas and audience](Personas%20and%20audience.md) - Describes the main user types for pool LPs and Credit Account operators, including what they care about and how they can lose money.
- [Benchmarks and tresholds for metrics](Benchmarks%20and%20tresholds%20for%20metrics.md) - Sets green/yellow/red operating thresholds for monitoring both pool positions and Credit Accounts.
- [Data requirements and to-dos](Data%20requirements%20and%20to-dos.md) - Lists the missing endpoints, feeds, components, and backend primitives needed to support the product vision.
